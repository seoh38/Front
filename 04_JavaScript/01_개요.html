자바스크립트 개요

자바스크립트

웹 브라우저 에서 많이 사용되는 인터 프리터방식의 스크립트 언어 (함수, 객체지향)
- 인터프리터 방식 : 컴파일 과정 없이 실행하며서 코드를 하나씩 읽어내는 방식. (코드에 문법 오류가 있으면 실행 시점에서 알 수 있다.) 컴파일 과정이 없기 때문에 실행 속도가 빠르다.

1. 자바 스크립트 사용법

  1. inline(인라인) : 태그 내에 직접 간단한 코드를 작성해서 실행하는 방법
  2. internal(내부) : 문서 내에 script 태그 영역에 소스코드를 박성해서 실행하는 방법
  3. external(외부) : 별도의 .js 파일을 작성해서 가져다가 실행하는 방법

1) inline 방식

태그내에 직접 간단한 소스코드를 작성해서 실행하는 방법
자바스크립트 소스가 몇줄 안되는 경우 inline 방식을 사용한다.

2) internal 방식

스크립트 태그 영역에 함수단위로 소스코드를 작성해놓고
이벤트 발생시 해당 함수 호출함으로써 실행하는 방식이다.

3) external 방식

별도의 .js 파일로 작성해서 가져다 사용하는 방법으로
자바 스크립트의 양이 많거나 중복되는 코드가 있을 경우 자바스크립트 코드 부분을 외부 파일로 작성한다.

2. 데이터 입출력

1) 데이터 입력

window.confirm()
- 어떤 질문에 대해 "예/아니요"의 결과를 얻을 때 사용한다.
- 확인 버튼과 취소버튼이 나타나며 확인버튼 클릭 시 true, 취소 버튼 클릭 시 false를 리턴한다.

window.prompt()
- 텍스트 필드, 확인, 취소 버튼이 있는 대화 상자를 출력한다.
- 입력한 메세지 내용을 리턴 값으로 돌려받는다.

2) 데이터 출력

document.write()
- 브라우저 화면 상에 페이지에 직접 값을 출력한다.

window.alert()
- 브라우저 화면창의 알림 창을 출력한다.

innerHTML로 요소 내용 변경

자바스크립트에서 태그 엘리먼트의 값을 읽거나 값을 변경할 때 inneHTML 속성을 사용한다.

console.log() - 개발자 도구 콘솔화면에 출력하고자 할 때 사용한다. (디버깅할 때 주로 사용)
